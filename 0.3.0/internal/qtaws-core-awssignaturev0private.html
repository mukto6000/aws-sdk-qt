void AwsSignatureV0Private::adornRequest(QNetworkRequest &request, const AwsAbstractCredentials &credentials) const

Adds AWS Signature Version 0 adornments to a AWS request.

In addition to service-specific request parameters, Amazon requires that version 1 signatures contain a number of common query parameters. This functions adds those query parameters to request if they're not already present.

The query parameters added by this function, as required by Amazon, are:

Parameter NameParameter Value
AWSAccessKeyIdThe AWS Access Key ID from credentials.
SignatureVersionThe version returned by AwsSignatureV0::version(); ie 0.
TimestampCurrent UTC timestamp in ISO 8601 format, eg 2013-10-30T12:34:56Z.

Note: If request includes an Expires parameter, then Timestamp is not added.

Note: The SignatureVersion header is optional for version 0 signatures, but this function always includes it for clarity.

See also AwsAbstractCredentials::accessKeyId().

[virtual] QByteArray AwsSignatureV0Private::canonicalQuery(const QUrlQuery &query) const

Returns the AWS Signature version 0 canonical query string for query.

This function returns a string containing the concatenation of Action and Timestamp (or Expires) query parameters.

For example, for the following SQS query string:

?Action=CreateQueue&QueueName=queue2&AWSAccessKeyId=AKIAIOSFODNN7EXAMPLE&SignatureVersion=1&Expires=2007-01-12T12:00:00Z&Version=2006-04-01

this function will return the following canonical form:

CreateQueue2007-01-12T12:00:00Z

Note: query must already contain an Action and either a Timestamp or Expires query paramter. See adornRequest().
