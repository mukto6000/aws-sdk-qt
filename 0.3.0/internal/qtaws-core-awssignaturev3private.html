[protected] QByteArray AwsSignatureV3Private::algorithmDesignation(const QCryptographicHash::Algorithm algorithm) const

Returns the AWS Signature version 3 algorithm designation for algorithm.

This function returns an algorithm designation, as defined by Amazon, for use with V3 signatures.

For example, if the algorith is `QCryptographicHash::Sha256`, this function will return `HmacSHA256`.

See also http://docs.aws.amazon.com/Route53/latest/DeveloperGuide/RESTAuthentication.html.

[protected] QByteArray AwsSignatureV3Private::authorizationHeaderValue(const AwsAbstractCredentials &credentials, const QNetworkAccessManager::Operation operation, QNetworkRequest &request, const QByteArray &payload) const

Returns the AWS Signature version 3 authorization header value for operation on request, with optional POST/ PUT payload, signed by credentials.

This function builds a V3 signature, and returns it to the caller. The returned header value is then suitable for adding as an `Authorization` header in the HTTP request, to be accepted by Amazon.

See also http://docs.aws.amazon.com/Route53/latest/DeveloperGuide/RESTAuthentication.html and setAuthorizationHeader.

[protected] QByteArray AwsSignatureV3Private::canonicalHeader(const QByteArray &headerName, const QByteArray &headerValue) const

Returns the AWS Signature version 3 canonical header string for the header headerName with value headerValue.

Note: Amazon documentation does not specify how to handle whitespace within quotes for V3 signatures, so here we use the same approach as V3 signatures. That is:

In canonical form, header name and value are combined with a single semi-colon separator, with all whitespace removed from both, _except_ for whitespace within double-quotes.

Note: This function is only applicable to the AWS3 format, not AWS3-HTTPS.

See also http://docs.aws.amazon.com/amazonswf/latest/developerguide/HMACAuth-swf.html, http://docs.aws.amazon.com/general/latest/gr/sigV4-create-canonical-request.html, and canonicalHeaders.

[protected] QByteArray AwsSignatureV3Private::canonicalHeaders(const QNetworkRequest &request, QByteArray * const signedHeaders) const

Returns the AWS Signature version 3 canonical headers string for request.

This function constructs a canonical string containing all of the headers in the given request.

Note: request will typically not include a Host header at this stage, however Qt will add an appropriate `Host` header when the request is performed. So, if request does not include a Host header yet, this function will include a derived Host header in the canonical headers to allow for it.

Note: This function is only applicable to the AWS3 format, not AWS3-HTTPS.

The signedHeaders parameter will be set to be semi-colon separated list of the names of all headers included in the result.

See also http://docs.aws.amazon.com/general/latest/gr/sigV3-create-canonical-request.html and canonicalHeader.

[protected] QByteArray AwsSignatureV3Private::canonicalRequest(const QNetworkAccessManager::Operation operation, const QNetworkRequest &request, const QByteArray &payload, QByteArray * const signedHeaders) const

Returns the AWS Signature version 3 canonical request string for operation on request, including optional POST / PUT payload.

Note, this function implments both AWS3 and AWS3-HTTPS variants of the AWS Signature version 3 - which are quite different.

The signedHeaders parameter will be set to be semi-colon separated list of the names of all headers included in the result.

See also http://docs.aws.amazon.com/amazonswf/latest/developerguide/HMACAuth-swf.html and http://docs.aws.amazon.com/Route53/latest/DeveloperGuide/RESTAuthentication.html.

[static protected] bool AwsSignatureV3Private::isHttps(const QNetworkRequest &request)

void AwsSignatureV3Private::setAuthorizationHeader(const AwsAbstractCredentials &credentials, const QNetworkAccessManager::Operation operation, QNetworkRequest &request, const QByteArray &payload) const

Sets the authorization header on request, for operation on the request with optional PUT / POST payload, signed with credentials.

This function will calculate the authorization header value and set it as the `Authorization` HTTP header on request.

See also http://docs.aws.amazon.com/general/latest/gr/sigV3-signed-request-examples.html and authorizationHeaderValue.

void AwsSignatureV3Private::setDateHeader(QNetworkRequest &request, const QDateTime &dateTime = QDateTime::currentDateTimeUtc()) const

Sets the AWS custom date header on request to UTC dateTime.

If request does not already contain an x-amz-date header, then this function will set a custom x-amz-date header to the value of dateTime formatted like "Fri, 09 Sep 2011 23:36:00 GMT".

Note: dateTime must be in UTC, that is QDateTime::timeSpec() must be Qt::UTC.
