Constructs an AwsSignatureV4Private object with hashAlgorithm, and public implementation q.

[protected] QByteArray AwsSignatureV4Private::algorithmDesignation(const QCryptographicHash::Algorithm algorithm) const

Returns the AWS Signature version 4 algorithm designation for algorithm.

This function returns an algorithm designation, as defined by Amazon, for use with V4 signatures.

For example, if the algorith is `QCryptographicHash::Sha256`, this function will return `AWS4-HMAC-SHA256`.

See also http://docs.aws.amazon.com/general/latest/gr/sigv4-create-string-to-sign.html.

[protected] QByteArray AwsSignatureV4Private::authorizationHeaderValue(const AwsAbstractCredentials &credentials, const QNetworkAccessManager::Operation operation, QNetworkRequest &request, const QByteArray &payload, const QDateTime &timestamp) const

Returns the AWS Signature version 3 authorization header value for operation on request, with optional POST/ PUT payload at timestamp, signed by credentials.

This function builds an V4 signature, and returns it to the caller. The returned header value is then suitable for adding as a `Authorization` header in the HTTP request, to be accepted by Amazon.

See also http://docs.aws.amazon.com/general/latest/gr/sigv4-signed-request-examples.html and setAuthorizationHeader.

[protected] QByteArray AwsSignatureV4Private::canonicalHeader(const QByteArray &headerName, const QByteArray &headerValue) const

Returns the AWS Signature version 4 canonical header string for the header headerName with value headerValue.

In canonical form, header name and value are combined with a single semi-colon separator, with all whitespace removed from both, except for whitespace within double-quotes.

See also http://docs.aws.amazon.com/general/latest/gr/sigv4-create-canonical-request.html and canonicalHeaders.

[protected] QByteArray AwsSignatureV4Private::canonicalHeaders(const QNetworkRequest &request, QByteArray * const signedHeaders) const

Returns the AWS Signature version 4 canonical headers string for request.

This function constructs a canonical string containing all of the headers in the given request.

Note: request will typically not include a Host header at this stage, however Qt will add an appropriate `Host` header when the request is performed. So, if request does not include a Host header yet, this function will include a derived Host header in the canonical headers to allow for it.

The signedHeaders parameter will be set to be semi-colon separated list of the names of all headers included in the result.

See also http://docs.aws.amazon.com/general/latest/gr/sigv4-create-canonical-request.html and canonicalHeader.

[protected] QByteArray AwsSignatureV4Private::canonicalRequest(const QNetworkAccessManager::Operation operation, const QNetworkRequest &request, const QByteArray &payload, QByteArray * const signedHeaders) const

Returns the AWS Signature version 4 canonical request string for operation on request, including optional POST / PUT payload.

The signedHeaders parameter will be set to be semi-colon separated list of the names of all headers included in the result.

See also http://docs.aws.amazon.com/general/latest/gr/sigv4-create-canonical-request.html.

[protected] QByteArray AwsSignatureV4Private::credentialScope(const QDate &date, const QString &region, const QString &service) const

Returns the AWS Signature version 4 credential scope for the given date, region and service combination.

void AwsSignatureV4Private::setAuthorizationHeader(const AwsAbstractCredentials &credentials, const QNetworkAccessManager::Operation operation, QNetworkRequest &request, const QByteArray &payload, const QDateTime &timestamp) const

Sets the authorization header on request, for operation on the request with optional PUT / POST payload, at timestamp, signed with credentials.

This function will calculate the authorization header value and set it as the Authorization HTTP header on request.

See also http://docs.aws.amazon.com/general/latest/gr/sigv4-signed-request-examples.html and authorizationHeaderValue.

QDateTime AwsSignatureV4Private::setDateHeader(QNetworkRequest &request, const QDateTime &dateTime = QDateTime::currentDateTimeUtc()) const

Sets the AWS custom date header on request to UTC dateTime.

This function will set a custom x-amz-date header to the value of dateTime, formatted to DateTimeFormat.

Note: Although Amazon labels this as a "date", it is in fact a full timestamp.

Returns dateTime as a convenience to callers.
